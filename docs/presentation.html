Intelligent Routing

Keyword Scoring Algorithm

Each agent declares a list of domain keywords. The router scores the query against every agent's keywords and dispatches to the highest scorer. Ties default to the Finance Q&A agent.

# router.py (simplified)
def route_query(self, question: str) -> str:
    scores = {}
    for name, agent in self.agent_registry.items():
        score = agent.can_handle(question)
        scores[name] = score

    best = max(scores, key=scores.get)
    return best  # e.g. "tax_education_agent"

Extensible Design

New agents are registered automatically. Subclass BaseAgent, implement can_handle() and process() â€” the router picks them up with zero config changes.

LangGraph Orchestration

StateGraph Workflow

Uses LangGraph's StateGraph â€” a directed acyclic graph of nodes and conditional edges. State flows immutably and each node can read/write a typed WorkflowState.

# orchestrator.py
graph = StateGraph(WorkflowState)
graph.add_node("router",        route_node)
graph.add_node("execute_agent", execute_node)
graph.add_node("finalize",      finalize_node)
graph.add_edge("router", "execute_agent")
graph.add_edge("execute_agent", "finalize")
graph.add_edge("finalize", END)

WorkflowState

class WorkflowState(TypedDict):
    question:      str
    routed_agent:  Optional[str]
    agent_output:  Optional[AgentOutput]
    messages:      List[AgentMessage]
    status:        AgentStatus
    metadata:      Dict[str, Any]

Why LangGraph?

  • Declarative graph definition â€” easy to add new nodes or edges
  • Built-in state management â€” no manual state passing
  • Conditional edges support complex branching workflows
  • Native LangSmith integration for tracing every run
  • Supports streaming outputs and human-in-the-loop pauses

Multi-Agent Ready

The orchestrator can be extended to support parallel agent execution, agent collaboration, and iterative refinement loops â€” just add nodes and conditional edges to the graph.
